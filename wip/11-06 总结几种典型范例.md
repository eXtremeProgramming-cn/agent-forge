我会给你介绍我们已经做了的几种MAS的情况，然后我们来看看如何用Cynefin模型识别一些典型范例（exemplars）。

首先我们的整个讨论范围我觉得大体来说都是“研究用MAS”这个范畴。可能不局限在“社会科学研究”，但是其实也差得不多，就它不是那种高度依赖实验数据的研究，你可以想想怎么定义这个范畴。

第一种情况是非常直白的，比如说我要搜集一些很直观的信息。比如什么呢？比如“下载News on China过去已经发布过的所有新闻期刊”（参见 https://dongshengnews.org/en/news-on-china/ ）。这种情况基本上就不需要什么设计，甚至可能都不用创建智能体系统，叫Claude Code去做，下载完以后放在哪个目录下就行了。

我觉得这种情况是属于Clear的。

第二种情况，就好像很典型的我会用AgentForge来生成的一个系统。它会包含明确的三个步骤：采集数据；分析数据；形成报告。比如说我用得比较顺的一个这样的系统就是我的“数字主权指数”（DSI）评估。这个指数体系包含4个维度、16项指标，我针对每个指标又细化了大约15~20个有针对性的问题。原则上来说，只要把这20个问题都以客观真实的信息和数据填上了，再采用我已经预先定义好的每个指标的评分规则，一条一条按照规则去对数据，对到几分就是几分。也就是说，这里要采集什么数据是很清晰的、怎么分析是很清晰的。虽然比较麻烦，且必须有一个分析过程，这几步做下来就能达成目的。我对金砖11国的DSI评估就是用这个过程完成的。

我觉得这种情况是属于Complicated的。

（这里有一点小问题我没太想清楚。像这里的“分析”也就是“评分”这个过程，它的独特之处是什么？是什么使它区别于前面的Clear的场景？我粗粗想一下觉得是AI的智能介入了，不过介入的又是规则相当清晰的，而不是需要深度总结、抽象思考、“灵光一闪”的那种智能介入——我认为那种介入至少在目前阶段来看必须由人来做。所以你帮我总结一下，是什么因素使这种场景区别于前面的Clear的场景了？）

第二种情况有一个变体。有些时候，我还不确定最终用于采集数据的非常明细的问题应该是什么，或者这些问题是随研究对象而有变化有定制的。比如我举个例子：我做一个区域国别研究的MAS。那么这里面有一项指标是“这个国家的农业生产情况”，那这一项指标就必须细化成很具体的问题才能采集数据，而且细化的问题是随国家而变的。比如对于尼泊尔，细化出来的问题肯定就应该有“水稻种植面积有多大”、“稻米年产量多少”；而对于加纳，问题就可能应该变成“可可豆的年产量多少”。那么为了得到这个颗粒度的有针对性的问题，我会先从比较粗的颗粒度上（仍然是针对这些指标）做一轮调研，我通常称之为overview，然后基于overview的成果再来针对这个国家、这个指标生成（formulate）一系列有针对性的细粒度问题，用来采集数据。

这样一来，这个多智能体系统的流程就成了：overview -> question formulation -> data collection -> analyze -> report。

但是我感觉这种情况还是属于Complicated的，主要原因是我认为这个过程中不需要人的智能介入。

我不是很确定这个想法对不对，也需要你帮我判断下。

第三种情况，就不一样了：在AI的流程中间有一个很明显的需要人脑介入的环节。我有两个这样的例子。

第一个，还是区域国别研究，但是更深刻，是做一个国家的阶级分析和矛盾分析（马克思主义意义上的）。那么这种研究就有一个很重要的特征：你对每个国家的分析，必须有一个适用这个国家的理论框架，你不能拿着一个通用的理论框架去套所有国家。比如我做DSI评估，那我不需要针对每个国家调整理论框架的，我就这16个指标几乎是客观的评估，评出来几分就是几分，然后我再分析这个国家为什么得分高为什么得分低。但是阶级分析和矛盾分析就不一样，你要分析南非的阶级问题，你就必须有一个理解南非的种族问题历史脉络和现状的理论框架，在这个基础上来分析，不然你就抓不到重点，得到一些毫无意义的东西——比如说分析半天南非的小资产阶级，然而小资产阶级在南非根本就不怎么存在，它的社会矛盾是围绕种族问题来展开的。类似的，你要分析印度的矛盾，你就得有一个理解种姓问题的理论框架；你要分析中东国家的矛盾，你就得有一个理解伊斯兰教的理论框架。不然这个分析根本就深入不下去。

那我目前做这种研究的方法是，我先用比较标准化的一套指标采集信息，包括采集这个国家所有左翼组织和思想家的分析文章，做一轮分析（我称之为conjunctural overview），然后我去读这个材料，对这个国家的矛盾形成一些初步的感觉。然后我要去跟专家交流，形成一种洞察，然后决定我用什么理论框架来指导对这个国家的分析。比如南非的情况，它可能是一种“新形态的殖民主义”理论框架。然后我想办法把这个指导性的理论框架导入到MAS中，然后剩下的问题才会转化成一个Complicated的问题。

第二个，是我做一个“News on China”的每周新闻摘要。它的制作过程分为两段，第一段是新闻故事的选取，我要决定从这周发生的所有新闻中，哪10~12条新闻要进入这个期刊；第二段是针对被选中的新闻线索（leads），进行全方位调研（从事件全貌、媒体报道、专家评论、历史背景、专业知识、地缘政治、影响分析这7个角度），然后根据调研结果再撰写80个单词以内的很精炼的摘要。我一开始是想把这整个流程几乎完全自动化，但我发现新闻线索的选取这个过程很难自动化，它就是需要很多人的介入。比如一些最常见的问题：这几条新闻线索其实是讲的同一件事（比如都是四中全会相关，只不过从人事、经济、科技等不同角度来报道），但AI识别不出；我用一个新闻线索但是我还想把另一个有点相关又不是那么完全相关的新闻拉进来一起讲个更完整的故事；哪种新闻更重要、为什么重要；一期邮件期刊里怎么平衡各类型新闻、正面负面新闻的比例；等等。这些问题，我觉得必须要编辑动脑子来决定，AI决定不好。所以我现在整个制作过程就分为完全分离的两段，摘要写作、编撰成邮件期刊这一步，完全不关心前面的推荐这一步是怎么发生的，两者完全解耦，反正编辑给出一个新闻线索的列表去进入第二步就行了。

我感觉这两个例子都是Complex的情况。我不确定我的感觉对不对，以及这个将Complex和Complicated区分开的关键特征是什么。你帮我也想想。

第四种情况我倒是比较清楚。比如说我要研究贵州榕江村超，数字技术是如何助力乡村振兴。那为了把这个研究做扎实，我首先得去做很具体的基层调研，比如一些我想调研到的问题：到底一个月有多少流量；这些流量是从哪些媒体/自媒体来的；县里是怎么运作这个庞大的自媒体矩阵的；这些流量又是怎么转化为乡镇的经济的；转化的形式有哪些、效果怎么样；各乡镇村寨做哪些工作来创造流量、来转化流量……就太多的具体情况我都不掌握，我还提不出理论性的问题。所以我需要先去做调研，去跟基层实际干事的人交流，在这个过程中再碰撞打磨出一个理论框架来理解和研究它。

我觉得这种情况是Chaotic。我觉得AI在这种情况下既不能代替我的分析，也不能代替我的信息采集（因为这些信息不存在于互联网上，没有档案记录，AI没地方去获得）。AI只能作为一种辅助工具，帮我做一些简单的辅助性工作，比如整理访谈录音之类的（这些是Clear的任务）。

请你帮我看看这个思路，提提你的想法？

我确实是想写一篇文章来讲这个主题。我觉得这篇文章有几个问题意识需要回应：

1. 什么是“研究用MAS”，它的定义、用途、边界是什么。这个概念需要有个澄清。很明显这种MAS跟一个——比如说——财务分析、市场分析的MAS是非常不一样的（当然也许以后会发现一些共通点，但不是我们现在的关注），所以先要说清楚我们讨论的范畴在哪里，我们这个讨论适用于什么，而不是泛泛地适用于“MAS”这一大类东西。

2. 怎么理解Cynefin模型在“研究”（我不确定哈，这个范畴的框定取决于上面问题1的结论）这个范畴里的投射关系——什么样的课题是Clear的，什么样的课题是Complicated的，等等。这些区分的依据是什么，怎么就能判断一个课题是Complex而不是Complicated（其余的类别也一样）。这种区分跟AI的关系是什么。具有什么属性的任务，AI就不适合了，就应该要人来介入——我觉得这是决定MAS的业务架构的一个关键问题。
    - “业务架构”是我临时想的一个词。比如说我做这个课题建的MAS分为3个阶段，信息采集、分析、生成报告。那么这里的“三个阶段”它是一种“架构”的概念，并且这个架构跟“技术架构”是不一样的。我之前的文章（declarative-multi-agent-architecture）主要就是在讨论技术架构。所以要讨论这个“三个阶段”，这肯定需要另一个术语、另一个概念。

3. 各个类别里，典型的应用场景是什么样的，典型的业务架构是什么样的——这时候我们就能得出一些具有典型性代表性的exemplars了。
    - 这一步就有点像在构建一个模式语言：这样的一个examplar，它适用于这个类别、这样的一种场景，为什么它适用。

4. 以后别人怎么用我们这个材料？
    - 他的应用场景如果跟我们这里已经归纳总结的场景恰好很相似甚至完全一样，他可以直接照搬一个exemplar去用——我们要想想这个“照搬”的机制是什么，是跟着这篇文章的介绍让AI去构建呢，还是另外再搞一组开源的模板
    - 他的应用场景如果跟我们归纳总结的不一样，没关系，他回到Cynefin模型去思考他自己的课题，根据我们识别出的关键属性，他要判断他这个课题术语哪个象限（哪个类别），这对他怎么设计他的MAS会有指导意义。
